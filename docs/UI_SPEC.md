# 「工业集成 (Industrial Integration)」UI 设计规范

**设计核心：** 强调“设备界面”的沉浸感。通过**斜切构图、结构化线条、以及模块化组件**模拟工业操控台。不再追求呼吸感，而是追求**功能密度**与**精密秩序**。

---

### 1. 结构与几何形态 (Structures & Geometry)

* **斜切与非对称：** 彻底打破矩形思维。全局按钮、标签页、进度条容器应引入 **45° 或 15° 的斜切角**。UI 边缘不再是圆润的，而是带有机械切割感的直线。
* **半透明蒙版层：** 容器背景使用高浓度的深灰/黑色半透明图层（`backdrop-blur-md`），通过背景模糊模拟“悬浮投影屏幕”的质感。
* **结构化装饰线：** 在容器边缘添加具有功能暗示的线条装饰（如：L型护角、点状矩阵、坐标轴线）。利用 `1px` 细线勾勒出模块之间的机械连接感。

### 2. 视觉层级与数据展示 (Data & Information)

* **高密度数值系统：** 核心数据不再仅仅是文字，而应被包裹在类似“传感器读数”的边框内。使用 **等宽字体 (Monospace)** 增强硬核技术感。
* **多层级标签 (Tags)：** 采用类似图片中“RANK 2”的模块化标签，背景色与文字对比强烈，利用带有几何图标的小色块来区分功能类别。
* **进度条逻辑：** 进度条应由多个小方块或分段线组成，模拟物理电量或压力表的机械反馈。

### 3. 色彩哲学：工业警示色 (Chromatic Strategy)

* **主色作为能量源：** 主色 `#CB81DA` 不再是装饰色，而是**“能量流”**。它应以长条光带、激活图标、或加粗路径的形式出现，代表系统当前的逻辑焦点。
* **工业黑灰基底：** 界面底色应使用带有一丝“冷调”的深灰色，而非纯黑。这能让 UI 看起来更像金属材质而非电子屏幕。
* **多维度指示色：** 使用极少量的薰衣草蓝 `#A1A7F6` 代表“技能/就绪”，亮绿色 `#4CAF50` 代表“正常/信号”。

### 4. 动态交互：机械响应 (Mechanical Interactions)

* **平移与滑动展开：** 避免简单的淡入淡出。组件进入时应伴随由中心向两侧水平拉伸，或沿 45° 斜线滑入的动效，模拟机械结构的伸缩。
* **故障感与粒子感：** 在关键切换瞬间，可以加入极细微的数字跳动或屏幕扫描线特效（Scanline），增强工业电子仪器的代入感。
* **悬停高亮：** 当鼠标悬停时，除了阴影，更重要的是**边框的亮化**和**内部图标的位移**，提供明确的“锁定”反馈。

## 🎨 核心色彩配置 (Color Palette)

基于 **OpenMCP 项目主题色** 与工业风格融合的配色方案：

| 类型 | 色值 (HEX) | 用途建议 |
| --- | --- | --- |
| **主色 (Primary)** | `#CB81DA` | **核心能量色**：高光点缀、激活路径、重要按钮、选中状态、链接。与项目 `--main-color` 一致。 |
| **主色悬停 (Primary Hover)** | `#A259B5` | **主色深态**：主色元素的悬停/按压反馈、滚动条悬停。 |
| **辅助主色 (Sub)** | `#2D323B` | **重工业底壳**：主容器背景、非对称侧边栏底色。与项目 `--main-dark-color` 一致。 |
| **功能副色 (Accent)** | `#A1A7F6` | **技能/精炼色**：成功状态、等级提升、特殊装备标识。与 OpenMCP 品牌渐变中的薰衣草色一致。 |
| **品牌强调 (Brand)** | `#5A00FF` | **深紫强调**：Logo、品牌标识、极高优先级提示。 |
| **中性文字 (Text)** | `#E0E0E0` | **主阅读色**：清晰的白色，用于重要数据和标题。 |
| **装饰灰 (Muted)** | `#5C5C5C` | **结构线条**：分割线、未解锁状态、坐标轴辅助线。 |
| **信号/正常 (Signal)** | `#4CAF50` | **状态指示**：成功、就绪、默认信号。与项目 `--signal-default-color` 一致。 |

### CSS 变量映射建议

```css
:root {
  --main-color: #CB81DA;
  --main-dark-color: #2D323B;
  --main-light-color: rgba(203, 129, 218, 0.7);
  --main-hover: #A259B5;
  --accent-color: #A1A7F6;
  --brand-accent: #5A00FF;
  --signal-default-color: #4CAF50;
}
```

## 近期 UI 修改历史与实现约定

以下为已落地的 UI 修改摘要及统一约定，新对话框与按钮组应与此保持一致。

### 1. 调试与主区域布局

| 项目 | 约定 |
|------|------|
| **工具调试 / 提示词调试** | 主区域内容最大宽度 **880px**、水平居中；splitter 拉满整宽，上下分区内部同样最大 880px 居中。 |
| **工具调试** | tree-select 左侧增加说明文案「选择需要调试的工具」（需 i18n）。 |
| **工具调试 JSON 输出** | 与提示词调试的 JSON 输出样式完全一致，不增加工具侧独有 wrapper 或样式。 |
| **调试入口文案** | 主入口为「选择下方模块开始调试」；引导文案中统一使用「提示词」而非「提词」。 |

### 2. 对话框 Footer 与按钮组（统一规范）

所有相关对话框的底部操作区采用同一套约定：

- **位置**：操作按钮放在对话框的 **footer** 插槽，不放在 body 内。
- **对齐**：footer 使用 `display: flex; justify-content: flex-end;`，按钮组**右对齐**。
- **按钮组**：使用 `el-button-group`，类名 **`executor-actions-group`**，视觉上为一组相连按钮（圆角 8px，首尾圆角，中间共边）。
- **按钮角色**：
  - 次要操作（取消、复制等）：类名 **`btn-secondary`**，边框与背景与主题一致，hover 时浅色背景。
  - 主要操作（确定、导出、提交等）：类名 **`btn-execute`**，使用 `--main-light-color-*` 系列，字重 600，hover 时主色加深。
- **可选**：若有「确认」类操作，可增加 Ctrl+Enter 快捷键说明（如 `CTRL` + 回车图标）。

**适用组件**：提示词调试弹窗、交互测试提词对话框、批量测试中的 Skill/预设类对话框、导出 MCP 配置对话框、Skill 路径对话框等。

### 3. 导出 MCP 配置对话框

- **标题与副标题**：使用自定义 `#header` 插槽；标题为主标题，其下为一行副标题（说明文案），副标题字号约 13px、**opacity: 0.5**。
- **Body**：说明段落、文件名输入行（label + input + .json + 「如何使用」链接）、预览区块（标题 + 可滚动代码区）；代码区使用等宽字体、边框与背景与主题一致。
- **Footer**：按钮组「复制」+「导出」，样式遵循上述 executor-actions-group / btn-secondary / btn-execute 约定。
- **预览更新**：对话框打开或导出数据变化时，通过 watch 刷新预览内容，避免连接/模型切换后预览不更新。

### 4. Skill 路径对话框

- Footer 与导出/提示词对话框一致：右对齐、`executor-actions-group`、取消 `btn-secondary`、确定 `btn-execute`。

### 5. 已移除的 UI

- **系统提示词入口**：  
  - **交互测试**：设置栏中已移除「系统提示词」按钮（即移除 `SystemPrompt` 组件）。  
  - **批量测试**：用例输入区工具栏中已移除系统提示词按钮及对应对话框，仅保留 SKILL 等其余能力。  

以上为近期 UI 修改的总结与可复用约定，新增或修改对话框时请参照第二节的 footer 与按钮组规范。
