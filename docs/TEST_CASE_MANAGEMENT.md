# MCP Tool 测试用例管理系统

## 功能概述

这是一个专门为 MCP Tool 设计的测试用例管理系统。它可以帮助你：
- 记录工具的输入和输出
- 保存和管理测试用例
- 批量运行测试并对比结果
- 从工具执行器快速生成测试用例

## 核心数据结构

### TestCase 接口

```typescript
interface TestCase {
    id: string;                      // 唯一标识符
    name: string;                    // 测试用例名称
    toolName: string;                // 工具名称
    description?: string;            // 描述
    input: Record<string, any>;      // 输入参数
    expectedOutput?: ToolCallResponse; // 预期输出（可选）
    actualOutput?: ToolCallResponse;   // 实际输出
    status?: 'pending' | 'passed' | 'failed' | 'running'; // 状态
    createdAt: number;               // 创建时间
    updatedAt: number;               // 更新时间
}
```

## 功能说明

### 1. 创建测试用例

有两种方式创建测试用例：

#### 方式一：手动创建
1. 在"测试用例"面板点击"创建测试用例"按钮
2. 填写测试用例信息：
   - 测试用例名称（必填）
   - 选择工具（必填）
   - 描述（可选）
   - 输入参数（JSON 格式）
3. 点击"保存"

#### 方式二：从执行器自动生成
1. 在"运行调试"面板执行一次工具调用
2. 执行成功后，点击"保存为测试用例"按钮
3. 系统会自动将当前的输入和输出保存为一个测试用例

### 2. 编辑测试用例

1. 在测试用例列表中找到要编辑的用例
2. 点击编辑按钮（笔形图标）
3. 修改测试用例信息
4. 点击"保存"

### 3. 运行测试用例

#### 单个运行
- 点击测试用例右侧的运行按钮（播放图标）
- 系统会调用对应的工具，并记录实际输出
- 如果设置了预期输出，会自动对比并显示测试状态

#### 批量运行
- 点击顶部的"运行全部测试"按钮
- 系统会依次执行所有测试用例
- 完成后显示统计结果（通过数/失败数）

### 4. 查看测试结果

1. 点击测试用例卡片可以查看详细信息
2. 结果对话框会显示：
   - 输入参数
   - 实际输出
   - 预期输出（如果有）

### 5. 删除测试用例

1. 点击测试用例右侧的删除按钮（垃圾桶图标）
2. 确认删除操作

## 测试状态说明

- **待执行 (pending)**: 测试用例已创建，但尚未执行
- **运行中 (running)**: 测试正在执行
- **通过 (passed)**: 测试执行成功
- **失败 (failed)**: 测试执行失败或输出与预期不符

## 使用技巧

### 1. 快速生成测试用例
在开发和调试过程中，每次成功执行工具后，可以立即点击"保存为测试用例"，快速积累测试用例库。

### 2. 从执行器复制参数
在创建或编辑测试用例时，可以点击"从执行器复制"按钮，快速复制当前运行调试面板中的参数。

### 3. JSON 格式化
输入参数支持 JSON 格式，可以点击"格式化 JSON"按钮自动美化代码。

### 4. 测试用例命名建议
- 使用描述性的名称，例如：`search_user_by_email_success`
- 包含测试场景和预期结果
- 使用下划线或驼峰命名法保持一致性

### 5. 持久化
测试用例保存在 `ToolStorage.testCases` 中，会随着项目一起保存。

## 未来扩展

以下功能可以在未来版本中实现：

1. **测试套件分组**: 将测试用例组织成套件
2. **导入/导出**: 支持导入导出测试用例为 JSON 文件
3. **高级断言**: 支持更复杂的输出对比逻辑
4. **测试报告**: 生成详细的测试报告
5. **参数化测试**: 支持数据驱动测试
6. **Mock 数据**: 集成更智能的测试数据生成
7. **CI/CD 集成**: 支持在持续集成环境中运行测试
